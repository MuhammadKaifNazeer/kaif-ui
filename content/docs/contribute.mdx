Thank you for your interest in contributing to Kaif UI! Your contributions are invaluable in helping us improve the library and make it a better tool for developers.

<Callout type="info">
## Contribution Rules   

- Follow the existing code style
- Write clear and concise comments
- Run linting and formatting tools
- Run the test suite to ensure your changes don't break existing functionality.
- Treat other contributors with respect and professionalism.
- Do not push directly to the main branch

These rules may be updated from time to time to reflect changes in the project or community.

</Callout>

## Get Started

<Steps>
    
<Step>

  ### Clone the respository

```bash
git clone https://github.com/MuhammadKaifNazeer/kaif-ui.git  
```
</Step>

<Step>

  ### Navigate to directory

```bash
cd kaif-ui
```
</Step>

<Step>

  ### Install Dependencies

```bash
npm install
```
</Step>

<Step>

  ### Run development server

```bash
npm run dev
```
</Step>

</Steps>

## FAQ

Some common questions you may encounter:

<Accordions>
<Accordion id="What kind of contributions can I make?" title="What kind of contributions can I make?">
You can contribute in several ways, including:

- Bug fixes and performance improvements.   
- New components, blocks, or templates.
- Improving documentation and guides.
- Writing unit tests to ensure the quality of the codebase.
- Suggesting new features or enhancements to existing components.
</Accordion>
    <Accordion id="How do I suggest a new feature or component?" title="How do I suggest a new feature or component?">
      Before working on a new feature or component, please open a GitHub issue to discuss your idea with the maintainers and community. This helps ensure your contribution aligns with the project's roadmap and style.
    </Accordion>
    <Accordion id="What should I do before submitting a pull request?" title="What should I do before submitting a pull request?">
Ensure the following before submitting a pull request:

- Follow the **Contribution Rules** outlined above.
- Ensure your code is well-documented and easy to understand.
- Make sure all linting, formatting, and tests pass without errors.
- Double-check the code against the project's style and formatting guide.
</Accordion>
    <Accordion id="Can I work on multiple issues at the same time?" title="Can I work on multiple issues at the same time?">
        Yes, but we encourage contributors to focus on one issue at a time to avoid confusion. If you're working on multiple issues, create separate branches for each contribution to keep things organized.
    </Accordion>
    <Accordion id="What should I do if I find a bug?" title="What should I do if I find a bug?">
If you encounter a bug, please check if it has already been reported. If not, you can open a new issue on GitHub with the following details:

- A brief description of the bug.
- Steps to reproduce it.
- Expected vs. actual behavior.
- Screenshots (if applicable).
</Accordion>
    <Accordion id="Can I contribute without writing code?" title="Can I contribute without writing code?">
Absolutely! You can help by:

- Reporting bugs.
- Improving documentation or guides.
- Participating in discussions about feature requests or improvements.
</Accordion>
    <Accordion id="How are contributors recognized?" title="How are contributors recognized?">
All contributors to Kaif UI are acknowledged in the Contributors section of the documentation and in the GitHub repository. We deeply appreciate every contribution, big or small.   
</Accordion>
</Accordions>
